Next.js Portfolio Theme - Inspired by VS Code

This portfolio template is built with Next.js and TypeScript, drawing inspiration from the look and feel of Visual Studio Code. It’s designed to be easily customizable and extendable, with a focus on clean code and modular structure.

Tools Used

  • TypeScript
  • Tailwind CSS
  • shadcn
  • React Context (for global state management)

Additional Libraries

  • React Syntax Highlighter
  • cmdk (dev dependency for shadcn command)
  • Framer Motion for animations
  • next-themes
  • Lucide for icons
  • react-github-calendar (for GitHub contributions)

Project Folder and File Structure

The application follows best practices, making customization easier. The folders and files are named intuitively, so you can easily understand their purpose.

  • components: Contains all the reusable components.
  • constants: Houses all the app-wide data and configurations.
  • layouts: Includes layout components used across different pages.
  • app: All the routes are inside this directory, with some grouped together for better organization.

Images

  • All images are placed inside the /public/images folder.
  • There is an additional file in /constants/images.ts that imports and exports images to simplify usage.

Constants

  • The constants folder contains all the data used across the app.
  • Proper naming conventions are followed to make customization easier.

Components

  • All reusable components are placed inside the components directory.
  • The /components/ui folder is used for shadcn components. You don’t need to modify these unless you want to change the shadcn components.

Context

  • The context folder contains a single React Context for state management.

Root Layout

  • layout.tsx is the entry point of the application.
  • This file is configured to render child routes within a specific layout that includes a navbar, two sidebars, and a main section.
  • Two wrappers are used: one combines the two sidebars (which need to be client-side rendered due to user interactions), and the other is a route wrapper, which is also a client component.

Home Page (page.tsx)

  • This is the root of the application, containing personal data of the developer.
  • You can easily modify the content inside or customize the layout as needed.

Modifications

GitHub Configuration
  • Open /app/github.page.tsx.
  • This page has two components. Open each component to find a username variable.
  • Change the username variable to your GitHub username.
Project Details
  • To add your projects, go to /app/projects/page.tsx.
  • In this file, you will find an import statement: import { projects } from "@/constants/projects";
  • Navigate to /constants/projects and add your project details.
Project Dropdown in Sidebar
  • There is another array exporting from /constants/projects/.
  • You will see an array named projectFiles that lists projects displayed in the sidebar under the Project tab.
  • Change the labels as per your project names, and you are good to go.
General Advice

Rest of the pages, components, and files are grouped and named for better understanding. Explore each route and check the imports and components rendering there to easily make modifications.
